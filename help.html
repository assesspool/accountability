1

Subject Record Bar

Manages the JSON file that stores all cycles for one subject. Always load or save here first.

First visit for a new subject
Fill the Study / Participant Header, complete the cycle, then click Save Record. A JSON file is downloaded named subjectID_protocol_acct.json. Save this file — it is the subject's record.
Subsequent visits
Click Load Record and select the subject's JSON file. The header populates automatically, cycle history appears below, and the cycle number increments. Fill the current cycle, then click + Save This Cycle.
Status indicator
Green = record loaded and saved. Amber ● = unsaved changes — click Save Record before closing.
File management
One JSON file per subject per protocol. Store files in a secure location accessible at clinic (e.g. encrypted drive, institutional cloud). No data is transmitted — everything stays on your device.

Study / Participant Header

Identifies the protocol, site, and subject. Set once; loaded automatically from subject record on subsequent visits. Shared across all cycles.

FieldRequiredWhat to Enter
Protocol # Optional
Sponsor-assigned protocol number.
e.g. ABC-2023-01
PI Optional
Principal Investigator last name.
Participant # Required
Subject ID as it appears in the regulatory binder and eCRF. Used as the filename for the subject's JSON record.
e.g. 001-001
Participant Initials Optional
Subject initials. Appears in cycle history header.
e.g. J.D.
Initial Total Daily Dose Optional
Starting total daily dose in mg. Reference only — does not affect calculations.
e.g. 300 mg
Initial Dosing Regimen Optional
Starting regimen as text. Reference only.
e.g. 150 mg BID
Visit Label Required
Visit identifier used to label this cycle in the history table and JSON record. Should match your protocol visit schedule.
e.g. C3D1
Visit Date Required
Date of the current clinic visit. Used to calculate cycle length against the previous visit date.
Prev. Visit Date Required
Date of the previous cycle visit. Combined with Visit Date to compute actual cycle length and flag against the ±day window.
⚠ Leave blank for Cycle 1 Day 1 (no prior visit).
Visit Window (±days) Required
Protocol-defined scheduling window in days. Default is ±3. Cycle length flags fire if actual days fall outside 28 ± this value.
e.g. 3 → acceptable range 25–31 days
Staff — Dispensing IP Required
Initials of staff member who dispensed the investigational product to the subject.
Staff — IP Accountability Required
Initials of staff member performing accountability reconciliation (may be the same person).
Dose Modification? Required
Select Yes if a dose change occurred this cycle. Reveals three additional fields: New Total Daily Dose, New Dosing Regimen, and Reason for Modification. Also triggers a DOSE MOD flag and generates an eCRF logline reference in the Pending eCRF Actions panel.
3

Dispensation — This Cycle

Records what was given to the subject at this visit. Drives all pill count calculations.

FieldRequiredWhat to Enter
Dose / Regimen Dispensed Required
Select the dose and frequency dispensed at this visit. Options are 50/100/150 mg in BID or QD. This sets pills-per-dose and doses-per-day used in all calculations. For dose reductions, select the new (lower) dose.
150mg BID = 3 pills × 2 doses = 6 pills/day
# Bottles / Vials Dispensed Required
Number of bottles handed to the subject. Changing this number automatically adds or removes bottle rows below. Each bottle holds 60 pills.
150mg BID for 28 days = 168 pills needed → 3 bottles (180 pills) dispensed
Total Pill Qty Delivered Calculated
Auto-calculated: bottles × 60. Read-only. This is the starting pill count for reconciliation.
Date & Time Dispensed Optional
Exact date and time IP was given to subject. Used for records; does not affect calculations. Relevant for late-dispense Day 1 events.
Bottle ID / UNK Required
Enter the bottle identifier printed on the label, or UNK if not available at time of entry. Sponsor requires tracking of which specific bottles were dispensed and returned. One row per bottle.
Return Status (per bottle) Required
Status of each bottle at return visit. Color-coded dot: green = returned, amber = pending, red = lost/missing.

Not Returned — bottle not yet back (normal at dispense visit).
Returned (partial) — returned with pills remaining inside.
Returned (empty) — returned empty, all pills accounted for or used.
Lost / Missing — triggers a red BOTTLE LOST flag; document and notify monitor.
Date Dispensed (per bottle) Optional
Date this specific bottle was given. Useful when bottles from different dispensation events are being reconciled together.
4

Pill Diary — Dosing Dates

Transcribed from the subject's paper pill diary. Defines the date range over which the subject was expected to take medication. Dose assumed uniform for the full cycle unless adjusted in Section 5.

FieldRequiredWhat to Enter
Dosing Start Date Required
First day the subject took medication this cycle. For Cycle 1 and subsequent cycles, this is typically the day of the clinic visit (the PM dose of Day 1 per protocol, as the AM dose is held until dispensation).
⚠ If dispensed late in the day, the AM dose may have been held — add a Late Dispense adjustment in Section 5.
Dosing End Date Required
Last day the subject took medication before this visit. Typically the day before the current clinic visit. Taken from the subject's diary.
Days on Drug Calculated
Auto-calculated: (End Date − Start Date) + 1, inclusive of both endpoints. This is the denominator basis for compliance.
Start 18-Mar, End 14-Apr → 28 days
Expected Pills Taken Calculated
Auto-calculated: Days × pills/day (from dose selection). This is the raw B value before adjustments. Any denominator adjustments from Section 5 are subtracted to produce the final adjusted expected.
28 days × 6 pills/day = 168 pills
5

Dose Adjustments & Interruptions

Captures individual dose events that change the expected pill count. Each event adjusts the reconciliation math and generates a pre-filled eCRF administration log logline reference. One row per event.

Event TypeImpacteCRF Loglines GeneratedNotes
Late Dispense — Day 1 Partial DENOM ADJ Note only — no 0-dose logline needed Pre-fills with pills-per-dose (one half of a BID day). Use when dispensed late and AM dose was held per protocol.
Vomited / Unable to Take DENOM ADJ Single 0-dose logline (start = end = event date) Protocol-defined — excused from compliance denominator.
Dropped / Unusable Pill DENOM ADJ Single 0-dose logline Pill cannot be retaken. Excused from compliance denominator.
Forgotten Dose COMPLIANCE MISS Single 0-dose logline Counts against compliance %. Diary should document reason.
Drug Held (AE / toxicity) INTERRUPTION 3 loglines: close active → open 0-dose → open restart Requires restart date to close the eCRF sequence. Open holds fire a persistent red flag.
Drug Held — Neutropenia (ANC) INTERRUPTION 3 loglines: close active → open 0-dose → open restart Common in oncology. Subject may restart between visits after lab confirmation. Enter restart date when known.
Drug Held — Surgery / Procedure INTERRUPTION 3 loglines: close active → open 0-dose → open restart Peri-operative hold. Restart date typically known in advance.
Drug Held — DLT Evaluation INTERRUPTION 3 loglines: close active → open 0-dose → open restart Dose-limiting toxicity evaluation hold. May lead to dose reduction on restart.
Dose Reduction DENOM ADJ 2 loglines: close prior dose line → open new dose line Permanent and unidirectional — dose cannot go back up. Select new dose at restart.
Site-Directed Hold INTERRUPTION 3 loglines: close active → open 0-dose → open restart Hold instructed by site staff (e.g. pre-procedure, safety concern).
Other User-selectable Single 0-dose logline Document reason in Note field. Select Denom Adj or Compliance Miss based on protocol guidance.
FieldWhat to Enter
Event Date
Date the event occurred. For holds, this is the first day drug was not taken. Used as the Start date in the eCRF 0-dose logline.
Pills Affected
For single-dose events (vomited, dropped, forgotten): number of pills missed for that event. For holds, leave blank — pill count is auto-calculated from hold dates × pills/day.
eCRF Action Taken
The Action Taken with Dose value for the eCRF exposure logline. Auto-set based on event type. Options: Drug Held / Drug Restarted / Dose Reduced / Dose Increased / Drug Withdrawn.
Impact on Compliance
Denominator adjust — pills subtracted from expected before compliance is calculated (event is excused). Compliance miss — pills stay in denominator; compliance % decreases.
Restart Date (holds only)
Date drug was restarted after a hold. Required to close the eCRF logline sequence. Until entered, an ⚠ OPEN HOLD badge pulses on the card and a red flag fires in the summary. The hold day count (restart − hold start) × pills/day is auto-subtracted from the denominator.
Restart Dose (holds only)
Dose on restart if different from prior (e.g. dose reduction on restart after DLT). Defaults to same dose if left blank.
Note / Reason
Free text reason. Maps directly to the Reason for Action Taken column in the eCRF exposure log. Also feeds the discrepancy notes in the print summary.
e.g. Grade 3 neutropenia, ANC < 1.0 × 10⁹/L
6

Drug Return & Diary Review

Records what came back from the subject at this visit. The total pills returned is the key input that drives compliance and delta calculations.

FieldRequiredWhat to Enter
Date Study Drug Returned Optional
Date bottles were physically returned. Usually the same as Visit Date. Record if different.
# Bottles / Vials Returned Required
Count of bottles returned. All dispensed bottles must be accounted for — whether full, partial, or empty. Update the individual bottle status rows in Section 3 accordingly.
Total # Pills / Vials Returned Required
Physical count of pills remaining across all returned bottles combined. This is the most critical entry — it drives the compliance calculation. Count carefully; this is the number written on the accountability log.
⚠ Any delta from expected triggers a flag immediately. Zero tolerance — any discrepancy is flagged.
Drug Diary Returned & Reviewed? Required
Confirms whether the subject's paper pill diary was collected and reviewed this visit. Select No to trigger a DIARY NOT REVIEWED flag.
Reasons for IP Discrepancies Conditional
Free text. Required when there is a pill count discrepancy or any red/amber flag. This text appears in the print summary and CSV export. Maps to the Comments column on the accountability form.
7

Reconciliation Summary

All values are auto-calculated. No entry required. Maps directly to the accountability log form columns.

Total Qty Delivered
= Bottles Dispensed × 60

B — Should Have Been Taken (adjusted)
= (Diary Days × Pills/day) − Denominator Adjustments

A — Total # Taken
= Total Qty Delivered − Total # Returned

Qty Expected Return
= Total Qty Delivered − B (adjusted expected)

Delta
= A − B (positive = excess taken, negative = deficit)

% Compliance
= A ÷ B × 100 (uses adjusted B, not raw diary expected)
Flags Reference
CYCLE LENGTH: Nd ✓
Actual cycle days within protocol window. No action needed.
PILL COUNT ✓
Actual return matches expected exactly (zero tolerance).
COMPLIANCE 100% ✓
All expected doses accounted for.
SHORT CYCLE: Nd
Cycle days below lower window bound. Verify dates.
EXCESS TAKEN: +N
Subject took more pills than expected. Verify diary; possible double-dose or counting error.
COMPLIANCE: N%
Compliance below 100% but above 80%. Document reason.
N BOTTLE(S) PENDING
One or more bottles not yet marked as returned. Update bottle status rows.
DIARY NOT REVIEWED
Drug diary not collected/reviewed. Required before finalizing accountability.
OPEN HOLD — [TYPE]
A drug hold has no restart date entered. eCRF logline sequence is incomplete. Enter restart date in Section 5.
LONG CYCLE: Nd
Cycle days exceed upper window bound. Verify dates; subject may have run short on medication.
DEFICIT: −N
Fewer pills returned than expected. Counsel subject. Document reason in IP Discrepancies field.
LOW COMPLIANCE: N%
Compliance below 80%. Review diary, counsel subject, document per protocol requirements.
BOTTLE LOST: ID
Bottle marked as lost/missing. Document and notify monitor.
INSUFFICIENT SUPPLY
Expected consumption exceeds dispensed quantity. Subject may have run out before visit.
N PILL(S) EXCUSED
Denominator adjustments applied. Compliance calculated on adjusted expected.
